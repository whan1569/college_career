## LG HelloVision VOD 데이터 기반 맞춤형 추천 플랫폼

### 프로젝트 링크

* [team3_web - 실제 서비스 구현 레포지토리](https://github.com/whan1569/team3_web)
  React/Vite 프론트엔드와 Node.js/Express 백엔드로 구현한 실제 VOD 추천 웹 서비스 레포지토리이다.
  사용자 인증, TOP20 VOD, 시청 이력, 장르 통계, 시청 패턴 분석, 유사 VOD 추천, 줄거리 기반 추천, 선호 장르 기반 추천 API가 포함되어 있다.

* [Team3 - 기획·분석·추천 전략 산출물 레포지토리](https://github.com/whan1569/Team3)
  LG HelloVision VOD 추천 프로젝트의 기획 자료, 데이터 분석 자료, 추천 전략 문서, 팀 산출물을 정리한 레포지토리이다.
  실제 VOD 시청 데이터 분석, TMDB 연동 전략, VectorDB 추천 전략, 줄거리 기반 추천 전략 등 추천 시스템 설계 과정이 포함되어 있다.

### 개요

LG HelloVision의 실제 VOD 시청 데이터를 활용하여 사용자 맞춤형 콘텐츠 추천 플랫폼을 구현한 팀 프로젝트이다.
프로젝트명은 VODiscovery이며, 사용자의 시청 이력과 콘텐츠 메타데이터를 기반으로 개인화된 VOD 추천, 실시간 인기 콘텐츠, 시청 패턴 분석 기능을 제공하는 웹 서비스 형태로 구현했다.

단순 더미 데이터 기반 프로젝트가 아니라 실제 기업 VOD 데이터를 활용했으며, AWS 환경에 배포하여 시연 당시 현장 참여자들이 직접 접속해 서비스를 사용해볼 수 있도록 구성했다.

### 추천 시스템 구조

```text
LG HelloVision VOD 시청 데이터
→ 사용자별 시청 이력 분석
→ 콘텐츠 메타데이터 전처리
→ 사용자 선호 장르 및 시청 패턴 분석
→ 콘텐츠 줄거리/메타데이터 임베딩
→ VectorDB 기반 유사 콘텐츠 검색
→ 사용자별 추천 결과 생성
→ 웹 서비스에서 추천 콘텐츠 노출
```

### 핵심 설계 포인트

이 프로젝트의 핵심 문제는 사용자 데이터가 충분하지 않은 초기 구간에서 추천 품질이 떨어지는 Cold Start 문제였다.
이를 해결하기 위해 사용자 행동 데이터에만 의존하지 않고, 콘텐츠 자체의 줄거리와 메타데이터를 벡터화하여 유사 콘텐츠를 추천하는 구조를 설계했다.

즉, 사용자의 시청 이력이 부족하더라도 콘텐츠의 의미적 유사도를 기반으로 추천 후보를 생성할 수 있도록 했고, 이후 사용자 시청 데이터가 누적되면 장르 선호도, 시청 패턴, 유사 사용자 기반 추천과 결합해 추천 품질을 높이는 방향으로 구성했다.

### 차별점

이 프로젝트는 일반적인 학습용 추천 시스템과 달리 실제 LG HelloVision VOD 데이터를 기반으로 구현했다.
실제 데이터는 콘텐츠 ID, 사용자 식별값, 시청 이력, 장르 정보, 외부 메타데이터 매핑 등에서 정제와 연결 문제가 발생하기 때문에, 단순 알고리즘 구현보다 데이터 전처리와 서비스 연결 구조가 중요했다.

또한 VectorDB를 활용하여 콘텐츠 임베딩 기반 추천 구조를 구성함으로써, 초기 사용자 데이터가 부족한 상황에서도 콘텐츠 기반 추천이 가능하도록 Cold Start 문제를 완화했다.

### 포트폴리오 요약

LG HelloVision의 실제 VOD 시청 데이터를 활용하여 개인화 콘텐츠 추천 플랫폼을 구현한 팀 프로젝트이다. React/Vite 기반 프론트엔드와 Node.js/Express/MySQL 기반 백엔드를 구성하고, JWT 인증을 통해 사용자별 시청 이력, 장르 통계, 시청 패턴, 맞춤형 추천 결과를 제공했다. VectorDB와 콘텐츠 임베딩을 활용해 사용자 데이터가 부족한 Cold Start 문제를 완화했으며, AWS에 서비스를 배포하여 시연 현장에서 참여자들이 직접 접속해 사용할 수 있도록 구현했다.

관련 레포지토리는 실제 서비스 구현 레포지토리인 `team3_web`과 기획·분석·추천 전략 산출물 레포지토리인 `Team3`로 구분된다.
