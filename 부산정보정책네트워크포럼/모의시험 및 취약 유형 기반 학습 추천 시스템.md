## TOPIK 모의시험 및 취약 유형 기반 학습 추천 시스템

### 개요

TOPIK 시험 대비를 위한 모의시험 웹 서비스를 기획하고, Node.js 기반 프로토타입을 구현했다.
사용자가 실제로 모의시험을 응시하면, 틀린 문제의 유형을 분석하고 자주 틀리는 유형과 유사한 문제를 다시 노출하는 **취약 유형 기반 학습 추천 구조**를 구현했다.

### 사용 기술

* JavaScript
* Node.js
* Express
* EJS
* MySQL
* express-session
* multer
* HTML/CSS
* 규칙 기반 추천 알고리즘
* 문제 유형 분류

### 핵심 아이디어

TOPIK은 1번부터 50번까지 문제 번호별로 출제 유형이 비교적 고정되어 있다는 점에 착안했다.
따라서 학습자의 데이터가 충분히 누적되지 않은 초기 상태에서도, 문제 번호와 문제 유형을 기준으로 학습 방향을 추천할 수 있다고 판단했다.

이 구조를 기반으로 사용자가 모의시험을 치르면 다음 흐름으로 추천이 동작하도록 설계했다.

```text
모의시험 응시
→ 정답/오답 판별
→ 틀린 문제 번호 확인
→ 문제 번호 기반 유형 분류
→ 자주 틀리는 유형 집계
→ 동일·유사 유형 문제 노출
→ 취약 유형 반복 학습
```

### 구현 내용

* Express 기반 웹 서버 구조 구성
* EJS 템플릿 기반 화면 렌더링
* 사용자 회원가입 및 로그인 기능 구현
* 세션 기반 사용자 상태 관리
* MySQL을 활용한 사용자, 문제, 응시 결과 데이터 관리
* TOPIK 문제 등록 및 문제 목록 조회 기능 구현
* 모의시험 응시 기능 구현
* 사용자의 답안과 정답을 비교하여 정답/오답 판별
* 틀린 문제의 문제 번호와 유형을 기록
* 사용자가 자주 틀리는 문제 유형을 파악
* 취약 유형과 동일하거나 유사한 유형의 문제를 다시 노출하는 추천 흐름 구현
* 문제 코드 체계를 통해 시험 회차, 문제 번호, 문제 유형을 구분
* 문제 번호 구간에 따른 TOPIK 문제 유형 타이틀 자동 매핑

### 기술적으로 의미 있었던 점

이 프로젝트의 핵심은 단순한 문제 풀이 서비스가 아니라, **응시 결과를 기반으로 학습자의 취약 유형을 추적하고 유사 문제를 재노출하는 피드백 루프**를 구현했다는 점이다.

일반적인 추천 시스템은 초기 사용자 데이터가 부족한 Cold Start 문제가 발생할 수 있다.
이 프로젝트에서는 TOPIK 시험의 고정된 문제 번호별 출제 유형을 활용하여, 데이터가 충분히 누적되지 않은 상태에서도 유형 기반 추천이 가능하도록 설계했다.

즉, 사용자 행동 데이터가 많지 않아도 문제 번호와 유형 정보를 기준으로 기본 추천을 제공하고, 이후 오답 기록이 누적되면 자주 틀리는 유형을 중심으로 추천 정확도를 높이는 구조를 구현했다.

### 프로젝트 성과

프로젝트 전체를 완성도 높은 서비스 수준까지 마무리하지는 못했지만, 핵심 기능인 **모의시험 응시 → 오답 유형 분석 → 취약 유형 기반 유사 문제 노출** 흐름은 구현에 성공했다.

또한 TOPIK 시험 구조를 활용한 추천 아이디어는 긍정적으로 평가받았고, 관련 포럼에 참석하는 경험으로 이어졌다.

### 프로젝트 한계

문제 데이터 구축, 유형 분류, 사용자 관리, 응시 결과 저장, 추천 알고리즘, 학습 콘텐츠 매핑까지 모두 구현해야 했기 때문에 개인이 제한된 시간 안에 완성하기에는 범위가 컸다.

특히 서비스 완성도를 높이기 위해서는 더 많은 문제 데이터, 유형별 학습 콘텐츠, 추천 결과 검증, UI/UX 개선이 필요했다.
다만 이 프로젝트를 통해 웹 서비스 구조, 데이터베이스 연동, 세션 기반 사용자 관리, 시험 문제 데이터 모델링, 취약 유형 기반 추천 시스템 설계를 경험했다.

### 포트폴리오 요약

Node.js, Express, EJS, MySQL을 사용해 TOPIK 모의시험 웹 서비스 프로토타입을 구현했다. 사용자가 모의시험을 응시하면 정답/오답을 판별하고, 틀린 문제의 유형을 분석해 자주 틀리는 유형과 유사한 문제를 다시 노출하는 취약 유형 기반 추천 흐름을 구현했다. TOPIK 문제 번호별 고정 출제 유형을 활용해 Cold Start 상황에서도 동작 가능한 규칙 기반 추천 구조를 설계했으며, 해당 아이디어를 긍정적으로 평가받아 관련 포럼에 참석했다.
